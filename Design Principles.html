Design Principles

Information Presentation

  • Use Columns and Rows to neatly present inforamtion.
  • Avoid clustering too much information together.
  • Prioritize information by making it bold.
  • Utilize spacing in order to present information neatly and improve readability.

Conceptuality

  • Simplify complex processes, breakdown complicated workflows into easy steps.
  • Visualize information effectively, apply diagrams, charts, and visual aids for abstract concepts
  • Abstain from using overly complicated sub menus and overcategorization.
  • Avoid the dangerouse assumption that the user will easily understand your system.

Design Basics

  • Ensure responsiveness in order to retain users and promote efficiency.
  • Maintain consistency by using an intuitive interface for your users.
  • Do design for accessibility to ensure all users can use your product.
  • Prioritize keeping the design consistent for the user experience.

Errors

  • Present clear error messages to reduce frustation for users.
  • Prevent using vague messages or something too technical.
  • Implement user feedback in order to quickly patch errors.

Skeuomorphism

  • Allow icons and buttons to mimic real life counter parts.
  • Don't design buttons that look odd or unintelligible.
  • Allow sublte shadows and gradients too create depth in your UI.

Intuitive Interface

  • Do provide clear error messages with actionable instructions.
  • Allow affordances and signifiers to guide your users.
  • Emphasize familiar UI patterns your user already understands.
